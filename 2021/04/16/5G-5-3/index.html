發表於 2021-04-16 更新於 2021-04-18 分類於 5G

一、从京东物流说起



在离用户比较近的地方设置分服务器
那和代理服务器 proxy 的区别是？

二、MEC 概念


Mobile Edge Computing 移动边缘计算 / Multi-access Edge Computing 多点边缘计算
在 5G 时代，数据吞吐量、用户密度、时延等指标要求非常高，云计算方式无法满足这些需求，需要引入边缘计算
网络边缘 = 靠近用户的位置

MEC 实现方式

5G 核心网用户面和控制面是分离的，所以 UPF 才可以提取出来放在边缘

三、MEC 架构


左边是 5G 核心网，右边是 MEC 架构（位于核心网和接入网之间，基于 NFV）
提供本地就近路由的分流，支持第三方应用的边缘应用环境
MEC 应用可以通过 MEP (MEC Platform) 发现服务成为消费者，自身也可以注册 MEC 服务的提供者
MEC System：提供业务编排，MEC 节点主机调用

四、MEC 部署方式


5G 透过 UPF 在网络边缘灵活部署达成分流，UPF 是由核心网的控制面统一管理
不同行业对边缘计算部署的位置要求不一样
区县级的：类似边缘云，可以给第三方提供边缘计算的服务
现场级的：一般是部署在运营商网络的接入点，和用户非常靠近

五、MEC 应用场景

1. 本地分流（最普遍的应用场景）

（1）企业园区：本地产生，本地终结，低时延高带宽，专用 LTE 无线网络
（2）校园：访问的内同相似性大，可以和第三方合作，例如行业咨询方案
（3）本地视频监控：会产生大量本地的回传流量，大部分的画面都是没有价值的

没有价值的画面保存在本地 MEC，有价值的才回传到服务器

（4）VR/AR：博物馆、体育赛事，视频的流量大
（5）本地视频直播：运营商可以提供一个 MEC 开放平台，提供视频直播应用，进行收入共享
（6）边缘 CDN：针对移动用户

2. 数据服务（基于 MEC 开放接口，第三方可以利用底层信息，提供精准的营销服务）

（1）室内定位：根据用户实时的位置，提供定制化的广告，或是互动式的服务
（2）车联网：实现自动驾驶，网络处理节点要离车辆很近

3. 业务优化

（1）视频 QoS 优化：不同用户需要的质量可能不一样
（2）视频直播和游戏加速
